请根据当前任务和代码变更，选择最小但有代表性的测试命令并执行。

$ARGUMENTS

执行步骤：

1. 先判断用户是否指定测试范围或命令。
2. 如果用户给出了明确命令，优先执行该命令。
3. 如果没有指定，检查项目结构和最近变更，选择最相关的单元测试、编译或 lint 命令。
4. 运行测试后，总结通过、失败和需要继续处理的点。

输出要求：

- 说明实际运行了哪些命令。
- 测试失败时，引用关键错误并给出下一步修复方向。
- 不要把大量日志原样贴出，只保留有诊断价值的内容。

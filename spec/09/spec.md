# LunaCode 斜杠命令注册与分发 Spec

## 背景

LunaCode 当前已经具备终端 TUI 对话、Agent Loop、上下文压缩、会话管理、记忆管理、权限模式和 Plan Mode 等能力。部分斜杠输入已经由对话入口中的分支逻辑处理，例如上下文压缩、会话、记忆、权限切换、取消运行以及计划/执行提示词改写；这些能力分散在不同判断中，缺少统一的命令元数据、帮助信息、别名冲突检测和补全体验。

下一阶段需要提供一套统一的斜杠命令注册和分发机制。用户输入以 `/` 开头时，系统优先按本地命令处理，常见操作不再交给 LLM 判断，从而减少 token 消耗、提升响应速度，并让清屏、查状态、切模式、查帮助等非对话操作具备确定行为。

本阶段只覆盖内置命令和命令基础设施，不开放用户自定义命令，不实现动态生成提示词，也不引入命令级权限控制；这些能力留给后续 Skill 系统。

## 目标

- LunaCode 提供统一的命令注册中心，集中管理命令元数据、别名和处理入口。
- LunaCode 能在启动阶段发现命令名或别名冲突，并以明确错误阻止应用继续运行。
- LunaCode 在用户回车入口区分斜杠命令和普通对话输入，斜杠命令走本地分发，普通输入继续交给 Agent。
- LunaCode 支持纯本地命令、影响界面状态的命令、向 Agent 发送预设提示词的命令三类执行模式。
- LunaCode 提供不绑定具体渲染实现的界面控制能力，让命令可以显示消息、发送用户消息、切换模式、查询状态并触发界面刷新。
- LunaCode 在状态栏和状态查询中明确展示 Agent 工作模式 `[DEFAULT]` 或 `[PLAN]`，并与权限模式共同呈现。
- LunaCode 支持斜杠命令别名和 Tab 补全，帮助用户快速发现和输入内置命令。
- LunaCode 内置高频命令：`/help`、`/compact`、`/clear`、`/plan`、`/do`、`/session`、`/memory`、`/permission`、`/status`、`/review`，并兼容 `/cancel` 和 `/permissions`。

## 功能需求

- F1: LunaCode 启动时注册所有内置命令。每条命令至少包含主名称、别名列表、简短描述、用法示例、命令类型、是否隐藏、参数提示和处理入口。
- F2: 命令主名称和别名都使用斜杠形式对外呈现，例如 `/help`、`/h`。注册和匹配时必须大小写不敏感。
- F3: LunaCode 在启动阶段检查所有命令主名称和别名。任意两个命令的主名称、别名或主名称与别名发生冲突时，应用必须以清晰的致命错误退出，不等到运行时再失败。
- F4: 命令解析器只处理以 `/` 开头的输入。空输入、空白输入和非 `/` 开头输入不进入命令解析。
- F5: 命令解析器将第一个空格前的内容识别为命令名，将其后的原始剩余内容识别为参数。参数保留用户输入的文本内容，但去掉首尾空白。
- F6: 命令解析器对命令名做小写归一化，因此 `/HELP`、`/Help` 和 `/help` 行为一致。
- F7: 当输入以 `/` 开头但命令未命中时，LunaCode 显示中文提示，指出未知命令，并引导用户使用 `/help` 查看可用命令。
- F8: 当用户回车提交输入时，LunaCode 先识别是否为斜杠命令。命令输入不作为普通用户消息写入对话；非命令输入才进入 Agent 对话流程。
- F9: `/cancel` 在任何状态下都优先执行，用于取消当前 Agent 响应、工具等待、权限确认或用户问题等待。
- F10: 除 `/cancel` 外，其他命令在 Agent 正在响应、工具正在运行、等待用户问题回答、等待权限确认或等待危险权限模式确认时，不执行命令逻辑，并显示“当前忙，请稍后再试”一类中文提示。此时普通输入仍用于回答问题或权限确认。
- F11: 命令按执行模式分为三类：纯本地命令只读取或展示本地状态；界面状态命令会影响 TUI、模式或状态栏；提示词命令会生成预设用户消息并交给 Agent。
- F12: LunaCode 提供命令可用的界面控制能力，至少支持显示本地消息、清理可见界面、发送用户消息给 Agent、切换 Agent 工作模式、查询 token 和运行状态、刷新状态栏。
- F13: `/help` 显示所有非隐藏命令的主名称、别名、简短描述和用法示例。
- F14: `/help <命令>` 显示单个命令详情。`<命令>` 可以是主名称或别名，命令不存在时显示中文错误并引导回 `/help`。
- F15: 隐藏命令不出现在 `/help` 列表中，也不参与 Tab 补全候选；但如果用户准确输入隐藏命令名称或别名，仍可正常执行。
- F16: `/clear` 只清空终端可见输出和当前输入行，不删除会话历史，不修改对话上下文，不影响后续 Agent 可见内容。
- F17: `/compact` 保持现有手动上下文压缩语义。空闲时触发手动压缩；忙碌或等待状态下提示稍后再试；完成后显示中文结果摘要。
- F18: `/plan` 只切换到 `[PLAN]` Agent 工作模式，不自动向 Agent 发送消息。进入 `[PLAN]` 时，权限模式同步切换为 `plan`，并记录进入前的权限模式。
- F19: `/do` 只切换回 `[DEFAULT]` Agent 工作模式，不自动向 Agent 发送消息。若用户在 `[PLAN]` 期间没有手动修改权限模式，`/do` 恢复进入 `/plan` 前记录的权限模式；若用户期间手动修改过权限模式，`/do` 尊重手动修改，不覆盖当前权限模式。
- F20: 状态栏显示当前 Agent 工作模式标记 `[DEFAULT]` 或 `[PLAN]`。当模式切换后，状态栏在下一次渲染中可观察到变化。
- F21: `/session` 保持现有会话命令语义：`/session [current|list|resume <id>|new]`。迁入注册中心后，原有查看当前会话、列出会话、新建会话、恢复会话和忙碌保护行为保持不变。
- F22: `/memory` 保持现有记忆命令语义：`/memory [list|delete <id>|on|off]`。迁入注册中心后，原有查看记忆状态、列出记忆、删除记忆、开关自动记忆和索引刷新行为保持不变。
- F23: `/permission` 保持现有权限命令语义。无参数时显示当前权限模式；一个参数时切换到 `default`、`acceptEdits`、`plan` 或 `bypassPermissions`；参数非法时显示中文用法提示。
- F24: `/permission bypassPermissions` 必须要求二次确认。用户确认后才切换到危险权限模式；用户拒绝或输入其他内容时取消切换，并保留原权限模式。
- F25: `/permissions` 作为 `/permission` 的兼容别名保留，行为与 `/permission` 完全一致。
- F26: `/status` 显示完整运行状态，包括当前 Agent 工作模式、权限模式、provider、model、累计输入 token、累计输出 token、会话短 ID、记忆开关、记忆最近状态、当前是否忙碌或等待。
- F27: `/review` 是提示词命令。执行时生成一段预设用户消息并交给 Agent，要求审查当前 `git diff` 中的代码变更，重点关注逻辑错误、安全问题、性能问题和代码风格。
- F28: `/review` 带参数时，将参数内容追加为“额外关注”并交给 Agent；不带参数时只发送默认审查提示词。
- F29: `/review` 生成的用户消息必须进入正常 Agent 对话流程，并像普通用户请求一样允许 Agent 使用工具查看当前工作区变更。
- F30: `/cancel` 注册进命令中心，作为独立命令可通过 `/cancel` 和 `/x` 执行；它不属于最初十个高频命令列表，但本阶段必须保持可用。
- F31: 内置短别名固定为：`/h` 和 `/?` 指向 `/help`，`/cl` 指向 `/clear`，`/cp` 指向 `/compact`，`/pl` 指向 `/plan`，`/d` 指向 `/do`，`/sess` 指向 `/session`，`/mem` 指向 `/memory`，`/perm` 和 `/permissions` 指向 `/permission`，`/st` 指向 `/status`，`/r` 指向 `/review`，`/x` 指向 `/cancel`。
- F32: Tab 补全只补全命令名，不补全参数。补全候选来自非隐藏命令的主名称和别名。
- F33: 用户在输入行中输入斜杠命令前缀并按 Tab 时，如果只有一个候选，LunaCode 自动补全该命令名。
- F34: 用户在输入行中输入斜杠命令前缀并按 Tab 时，如果存在多个候选，LunaCode 显示临时候选菜单。候选菜单不进入会话历史。
- F35: 临时候选菜单会在用户继续输入、再次按 Tab、提交输入、取消输入或候选失效时清除，并重绘当前输入行，避免终端输出堆积。
- F36: 命令注册中心允许登记隐藏命令，但本阶段不要求新增用户可见的隐藏命令。

## 非功能需求

- N1: 命令分发应足够轻量。纯本地命令和界面状态命令不应触发模型请求，也不应向对话历史追加普通用户消息。
- N2: 命令匹配、别名冲突检测和补全候选应具备确定性，同一组命令注册结果在每次启动中顺序一致、结果一致。
- N3: 命令处理不应直接绑定具体终端渲染实现。未来更换 TUI、增加测试界面或接入其他前端时，命令核心行为可以复用。
- N4: 用户可见提示、错误、帮助和状态输出使用中文，命令名称、模式名、provider、model、token 等技术标识保持原样。
- N5: 命令执行不能绕过现有权限安全策略。除本阶段明确保留的权限模式切换命令外，不新增命令级权限控制或特权后门。
- N6: `bypassPermissions` 相关提示必须明确危险性，且不能因为别名或补全路径绕过二次确认。
- N7: 命令补全和临时候选菜单不能破坏正在输入的内容、光标位置和后续回车提交行为。
- N8: 命令迁移后，现有 `/compact`、`/session`、`/memory`、`/permissions`、`/cancel` 的用户可见行为保持兼容。
- N9: 本阶段应包含自动化测试覆盖命令解析、注册冲突、别名匹配、忙碌状态优先级、内置命令分发、模式切换、帮助输出、补全候选和提示词命令。

## 不做的事

- 本阶段不支持用户自定义命令。
- 本阶段不支持从配置文件、Skill 或远端服务动态注册命令。
- 本阶段不实现动态生成提示词模板；提示词型命令只使用内置固定模板和用户参数拼接。
- 本阶段不实现命令级权限控制。
- 本阶段不补全命令参数。
- 本阶段不实现可用方向键选择并回车确认的完整交互式补全菜单。
- 本阶段不重新设计 `/session` 和 `/memory` 的子命令语义。
- 本阶段不清空或重写会话历史，`/clear` 只影响终端可见输出。
- 本阶段不把 `/review` 做成独立静态分析器；它只生成预设提示词并交给 Agent。
- 本阶段不修改 Skill 系统，也不把命令能力暴露为用户可编辑 Skill。

## 验收标准

- AC1: 启动时注册两条名称或别名冲突的命令，LunaCode 以清晰致命错误退出；注册无冲突命令时可正常启动。
- AC2: 输入 `/HELP test`、`/help test` 和 `/Help test` 时，命令名匹配一致，参数均被识别为 `test`。
- AC3: 输入非 `/` 开头的普通消息时，LunaCode 将消息送入 Agent；输入已注册斜杠命令时，不把命令文本当作普通用户消息交给 Agent。
- AC4: 输入未知斜杠命令时，LunaCode 显示中文未知命令提示，并包含 `/help` 引导。
- AC5: Agent 忙碌或等待用户/权限输入时，输入 `/status` 等非 `/cancel` 命令会得到稍后再试提示；输入 `/cancel` 能取消当前等待或运行。
- AC6: `/help` 输出包含所有非隐藏内置命令的主名、别名、描述和用法；`/help review` 输出 `/review` 的详情；隐藏命令不出现在列表中。
- AC7: `/clear` 执行后终端可见输出和当前输入行被清空，但会话历史仍可用于后续 Agent 上下文。
- AC8: 空闲状态执行 `/compact` 会触发手动上下文压缩并显示中文结果摘要；忙碌状态执行 `/compact` 会提示稍后再试。
- AC9: 执行 `/plan` 后，状态栏显示 `[PLAN]`，权限模式切到 `plan`，并且不向 Agent 发送预设消息。
- AC10: 执行 `/do` 后，状态栏显示 `[DEFAULT]`；如果 `[PLAN]` 期间未手动修改权限模式，权限模式恢复为进入 `/plan` 前的值。
- AC11: 在 `[PLAN]` 期间手动执行 `/permission acceptEdits` 后再执行 `/do`，Agent 工作模式回到 `[DEFAULT]`，权限模式仍保持 `acceptEdits`。
- AC12: `/session current`、`/session list`、`/session new`、`/session resume <id>` 的行为与迁移前一致，并在忙碌状态下保护会话切换类操作。
- AC13: `/memory`、`/memory list`、`/memory on`、`/memory off`、`/memory delete <id>` 的行为与迁移前一致。
- AC14: `/permission` 显示当前权限模式；`/permission default` 能切换到默认权限；`/permissions default` 与 `/permission default` 行为一致。
- AC15: `/permission bypassPermissions` 显示危险确认提示；确认后才切换，拒绝后保持原权限模式。
- AC16: `/status` 输出包含 Agent 工作模式、权限模式、provider、model、输入/输出 token、会话短 ID、记忆状态和当前运行状态。
- AC17: `/review` 发送给 Agent 的用户消息包含“请审查当前 git diff 中的代码变更”，并包含逻辑错误、安全问题、性能问题和代码风格四个关注点。
- AC18: `/review 并重点看异常处理` 发送给 Agent 的用户消息额外包含“额外关注：并重点看异常处理”。
- AC19: 所有固定短别名都能执行对应主命令，且帮助和补全能展示这些别名。
- AC20: 输入 `/pe` 后按 Tab 能单匹配补全为 `/permission` 或唯一匹配项；输入 `/p` 后按 Tab 能显示包含 `/plan`、`/permission` 等候选的临时菜单。
- AC21: 多匹配候选菜单出现后，继续输入、再次按 Tab、回车提交或取消输入时，候选菜单被清除，输入行被正确重绘。
- AC22: 项目编译通过，命令解析、命令注册、命令分发、内置命令、补全和模式联动相关自动化测试通过。

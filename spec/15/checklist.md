# LunaCode Claude 风格 TUI 视觉升级 Checklist

> 每一项都必须通过运行代码、自动化测试或终端观察验证。只有所有适用条目通过，才认为 spec/15 完成。

## 品牌、布局与视觉层级

- [ ] LunaCode 启动后显示紧凑品牌卡片，卡片包含品牌标识、版本或 `dev` 构建标识、当前模型、当前工作目录和快捷操作提示。（验证：在正常宽度终端启动程序，逐项核对卡片内容；对应 AC1）
- [ ] 启动卡片之后立即出现可操作的输入区域，整个界面保持滚动式对话记录，不进入固定分栏或全屏 Dashboard。（验证：启动后输入两轮消息并使用终端原生滚动查看历史；对应 AC1）
- [ ] 正常宽度下品牌卡片保持紧凑，窄终端下能缩短目录或切换精简布局，不覆盖输入行。（验证：分别以正常宽度和窄宽度运行或使用布局测试模拟；对应 AC1、AC15）
- [ ] 用户消息和助手消息具有不同的稳定符号与 Luna 月夜配色，不再把 `[complete]`、`[streaming]` 作为主要标识。（验证：提交普通问题并观察用户消息、流式回复和完成后的历史；对应 AC2）
- [ ] 品牌和主要交互使用紫色/蓝色，工具活动使用青色，成功、警告、错误分别使用绿、黄、红色，次要信息视觉弱化。（验证：运行涵盖工具成功、警告和错误的终端场景并目视对比；对应 AC2）
- [ ] 所有成功、失败、警告、问题和权限状态同时具有符号或明确文本，不依赖颜色才能理解。（验证：比较正常颜色与 `NO_COLOR` 输出；对应 AC3）

## 加载动画与活动状态

- [ ] 用户提交消息后、模型首个文本尚未到达时，界面显示会变化的 Spinner、当前模型动作、递增耗时和 Esc 取消提示。（验证：使用可控延迟 Provider 的自动化测试，并在 tmux 真实请求中观察；对应 AC4）
- [ ] 没有任何新模型事件到达时，Spinner 仍按稳定节奏更新，耗时不会因普通状态重绘而归零。（验证：使用固定/推进 Clock 连续生成多帧并比较帧与 elapsed；对应 AC4、AC19）
- [ ] 首个流式文本到达后，模型等待 Spinner 及时停止，助手正文继续按增量出现而不是整段一次输出。（验证：使用分段流式响应测试并观察终端输出；对应 AC4）
- [ ] 工具执行期间显示工具 Spinner、工具名、安全目标摘要、耗时和取消提示，不显示 `tool_running` 内部状态名。（验证：执行 ReadFile、Grep 和 Bash 测试请求；对应 AC5）
- [ ] 上下文压缩期间显示明确的压缩动作和动画，完成或失败后动画停止并显示对应结果。（验证：触发 `/compact` 或压缩事件集成测试；对应 AC5）
- [ ] 前台空闲且存在后台任务时显示后台任务数量、摘要和持续耗时；后台任务全部完成后动画停止。（验证：启动一个后台子 Agent，观察开始、运行和完成通知；对应 AC5）
- [ ] 前台工具、压缩、模型和后台任务同时有状态时，界面按既定优先级展示主要活动，状态切换不丢失工具完成记录。（验证：使用活动跟踪测试构造重叠事件；对应 AC5、AC17）
- [ ] 连续观察多个动画帧时，终端历史不会逐帧增加新行，动画始终在同一临时行更新。（验证：捕获多帧内存输出并在 tmux 中观察 scrollback；对应 AC6）
- [ ] 状态文案由长变短、动画结束或切换动作后，临时行没有旧字符、重复 Spinner 或可见控制码残留。（验证：运行实时区域长短帧测试并观察真实终端；对应 AC6）
- [ ] 活动完成、失败、取消、等待用户或等待权限后，Ticker 立即停止，不继续进行不可见的后台刷新。（验证：使用可控 Ticker 统计 stop 后的回调次数；对应 AC4、AC10、AC19）
- [ ] 动画线程使用低频定时调度而非忙等待，空闲时没有持续 tick，程序关闭后不会因动画线程无法退出。（验证：运行 Ticker 生命周期测试并观察测试 JVM 正常结束；对应 AC17、AC19）

## 工具生命周期记录

- [ ] 工具开始时的运行记录包含工具名和适合该工具的目标摘要，例如文件路径、搜索模式或截断命令。（验证：分别触发文件、搜索、命令和未知工具展示测试；对应 AC5、AC7）
- [ ] 工具成功后运行行转化为包含成功符号、工具名、目标摘要和实际耗时的紧凑历史记录。（验证：执行一次成功 ReadFile 并核对完成行；对应 AC7）
- [ ] 成功工具记录不包含完整 JSON 参数、完整文件内容或完整工具结果，但模型仍能基于完整结果生成最终回答。（验证：让 ReadFile 返回明显长内容，检查 UI 与最终回答；对应 AC7）
- [ ] 工具失败后 Spinner 停止并留下失败符号、工具名和可理解的截断错误摘要，随后输入仍可继续使用。（验证：调用一个必然失败的文件或命令操作；对应 AC8）
- [ ] 多个并行工具乱序完成时，每个 requestId 只生成一条对应的最终记录，不重复、不遗漏、不串用耗时。（验证：运行并行工具活动测试；对应 AC7、AC8、AC17）

## 紧凑状态、输入与命令补全

- [ ] 空闲输入区附近只常驻展示当前模型、Agent 模式、权限模式和会话标识，缺失字段被省略。（验证：构造完整状态和缺字段状态分别渲染；对应 AC9）
- [ ] 空闲紧凑状态不常驻展示 Provider、输入/输出 Token 和 Memory 状态。（验证：检查正常输入区域输出；对应 AC9）
- [x] 执行 `/status` 后仍能查看 Provider、模型、Token、会话、Memory 和运行状态等原有详细信息。（验证：在真实 TUI 输入 `/status` 并核对输出；对应 AC9、AC18）
- [ ] 等待用户回答时显示清晰问题提示且没有 Spinner，用户回答后流程可以继续。（验证：在 Plan Mode 触发 AskUserQuestion 或使用状态集成测试；对应 AC10）
- [ ] 等待权限确认时显示权限符号、权限模式和确认文案且没有 Spinner，允许或拒绝后提示正确结束。（验证：触发写文件或 Bash 权限确认并分别允许、拒绝；对应 AC10）
- [ ] 已取消、警告、错误和普通信息使用不同符号与语义色，同一个最终状态不会重复打印。（验证：逐个构造状态并重复 requestRender；对应 AC10）
- [ ] 动画更新期间输入一段中英文混合文本并移动光标，动画结束后内容和光标位置与操作结果一致。（验证：使用可控动画集成测试，并在 tmux 中手动输入；对应 AC11）
- [ ] 助手流式输出期间输入缓冲不会被清空或修改，回复完成后输入内容重新出现并可正确提交。（验证：在延迟流式回复期间预先输入下一条消息；对应 AC11）
- [ ] 长输入超过终端宽度时显示围绕光标的水平窗口，Home、End、左右移动、Backspace 和 Delete 仍作用于正确字符。（验证：在窄终端输入长中文和 emoji 文本；对应 AC11、AC15）
- [ ] 输入斜杠命令并按 Tab 出现多个候选时，命令和别名归属具有清晰层级，候选按宽度安全换行。（验证：输入可产生多个候选的前缀并调整终端宽度；对应 AC12、AC15）
- [ ] 选择单候选、关闭多候选或继续输入后，候选行被完整清除，输入文本和光标正确恢复。（验证：依次测试单候选、多候选和取消候选；对应 AC12）
- [ ] Spinner 更新与补全候选同时存在时，候选列表和输入光标不会闪烁、丢失或重复。（验证：在可控活动状态中触发命令补全；对应 AC11、AC12、AC17）

## 清屏、宽度与终端降级

- [x] 执行 `/clear` 后旧对话、旧工具记录、旧动画和旧状态不再出现，品牌卡片、当前紧凑上下文和输入区域重新显示。（验证：先产生多类输出再执行 `/clear`；对应 AC13）
- [ ] 清屏后提交新消息，用户消息、Spinner、工具记录和回复仍能正常显示，不因旧渲染缓存而漏绘。（验证：`/clear` 后执行一次真实工具请求；对应 AC13）
- [ ] 设置 `NO_COLOR` 后启动程序，输出不包含颜色 ANSI 序列，所有状态仍能通过 ASCII 符号或文本区分。（验证：以 `NO_COLOR=1` 运行并检查捕获输出；对应 AC3、AC14）
- [ ] 模拟 dumb、非交互或不支持 Unicode 的终端时，界面使用 ASCII Spinner、ASCII 成败标识和纯文本样式，提交、取消和补全仍可用。（验证：运行 TerminalProfile 与降级 TUI 测试；对应 AC14）
- [ ] ASCII 和 Unicode 两种模式中的 Spinner、成功、失败、警告、问题和权限标识语义一一对应。（验证：运行主题符号参数化测试；对应 AC3、AC14）
- [ ] 中文、emoji 和中英文混排的截断位置合法，不产生半个代理对、乱码或错误光标列数。（验证：运行列宽与截断自动化测试；对应 AC15、AC19）
- [ ] 在窄终端中，长模型名、长会话 ID、长工具目标和多候选不会覆盖输入行，优先裁剪次要信息。（验证：以多种小宽度运行布局测试；对应 AC15）
- [ ] 运行中调整终端宽度后，下一帧按新宽度重排，旧行被完整清理且输入仍可编辑。（验证：在 tmux 中调整 pane 宽度并观察动画和补全；对应 AC15）

## 生命周期、安全与架构集成

- [ ] 所有着色片段都正确重置样式，普通消息之后的文本和退出后的 Shell 提示符不会继承 LunaCode 颜色。（验证：检查捕获输出中的 reset，并在真实终端退出后输入普通命令；对应 AC16）
- [ ] 正常退出后恢复终端 raw mode、光标和输入回显，Shell 可立即正常使用。（验证：正常退出 TUI 后输入一条 Shell 命令；对应 AC16）
- [ ] 空闲时按 Esc 退出、运行时按 Esc 取消、异常启动或渲染失败三种路径都执行统一终端清理。（验证：分别运行退出、取消和异常生命周期测试；对应 AC16）
- [ ] Agent 事件观察者收到工具 requestId、Duration 和压缩事件，关闭订阅后不再收到事件。（验证：运行编排器事件观察测试；对应 AC5、AC7、AC19）
- [ ] 单个 UI 观察者抛出异常时 Agent Loop、状态更新和其他观察者仍正常运行。（验证：注册故障观察者并断言后续事件和循环完成；对应 AC17、AC18）
- [ ] 状态快照能够稳定列出运行中的后台任务，并在任务完成后移除，不向 TUI 泄漏后台管理器内部可变对象。（验证：运行后台状态集成测试；对应 AC5、AC19）
- [ ] 动画 tick、Agent 事件、流式文本和键盘输入并发发生时，捕获输出中没有交错半截 ANSI、重复正文或终端状态损坏。（验证：运行并发渲染测试并重复执行多次；对应 AC17）
- [ ] UI 升级前后的模型消息、工具调用参数、工具完整结果、权限决定和会话持久化内容保持一致。（验证：运行 Agent Loop、工具、权限和会话回归测试；对应 AC18）
- [ ] TUI 的自动化测试使用固定 Clock、可控 Ticker、显式 TerminalProfile 和内存输出，不需要真实模型或长时间 sleep。（验证：检查并运行 TUI 测试，确认执行时间稳定；对应 AC19）

## tmux 端到端场景

- [ ] 场景一——正常工具流程：在 tmux 中启动 LunaCode，确认品牌卡片和紧凑状态；输入“阅读当前项目的 pom.xml 和 TUI 入口，概括使用的终端依赖”；观察模型等待 Spinner、至少两个紧凑工具记录、流式中文回复以及最终输入恢复。（对应 AC1、AC2、AC4–AC7、AC9、AC20）
- [ ] 场景二——输入并发：模型或工具动画运行时预先输入 `/status`，移动光标并编辑；任务完成后确认输入未丢失，提交后详细状态正常显示。（对应 AC9、AC11、AC17、AC20）
- [ ] 场景三——权限与取消：请求执行一个需要权限确认的安全命令，观察权限提示没有 Spinner；拒绝后再次发起长任务并按 Esc，观察取消状态、动画停止和输入恢复。（对应 AC8、AC10、AC16、AC20）
- [ ] 场景四——补全与清屏：输入斜杠前缀触发多个候选，确认主题和换行；关闭候选后执行 `/clear`，确认品牌和输入区域重绘并可继续对话。（对应 AC12、AC13、AC20）
- [ ] 场景五——压缩与后台：触发一次上下文压缩并启动一个后台子 Agent，分别观察压缩动画、后台动画和完成通知；前台操作不被后台动画阻塞。（对应 AC5、AC17、AC20）
- [ ] 场景六——降级模式：使用 `NO_COLOR=1` 在新的 tmux pane 中启动 LunaCode，重复一次工具请求，确认纯文本 ASCII 动画、角色和工具结果均可理解且没有 ANSI 乱码。（对应 AC3、AC14、AC20）
- [ ] 场景七——宽度变化：在动画、长输入和补全候选出现时缩窄并恢复 tmux pane，确认实时区域重排、历史不被逐帧污染、输入和光标保持正确。（对应 AC6、AC11、AC12、AC15、AC17、AC20）

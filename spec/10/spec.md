# Skill 系统 Spec

## 背景

LunaCode 已经具备斜杠命令、工具调用、项目指令、会话和上下文管理能力，但可复用的 AI 操作仍主要依赖用户反复输入提示词，或写死在少量内置命令里。用户需要一种轻量方式，把常用 AI 工作流沉淀成可复用能力：既能通过斜杠命令显式调用，也能在普通对话中由 Agent 按需发现并加载。

Skill 系统用于解决这个问题。一个 Skill 是一份带元信息的 Markdown SOP 指令，也可以是一个带附属资源的目录。LunaCode 在上下文里只暴露 Skill 的名称和一句说明，真正需要时才加载完整指令，避免启动时和每轮对话浪费 token。

## 目标

- 让用户把可复用 AI 操作封装成独立 Skill，并通过 `/name` 形式调用。
- 支持项目级、用户级、内置级三层 Skill 来源，项目级覆盖用户级，用户级覆盖内置级。
- 支持两阶段加载：默认只暴露 Skill 名称和说明，按需加载完整 SOP。
- 支持 `inline` 和 `fork` 两种执行模式，满足轻量提示词复用和隔离执行两类场景。
- 支持 `$ARGUMENTS` 占位符，把用户调用 Skill 时传入的参数替换进 prompt body。
- 支持按 Skill 临时收窄工具白名单、临时指定模型，并在调用结束后恢复默认行为。
- 支持目录型 Skill 携带示例、模板、脚本和参考文档等附属资源，但不自动把这些资源塞进上下文。
- 支持执行时热更新 Skill 文件，解析失败时尽量回退到上一次成功解析的缓存版本。

## 功能需求

- F1: Skill 使用 YAML frontmatter 加 Markdown 正文定义。`name` 和 `description` 必填；`tools`、`mode`、`context`、`model` 可选。
- F2: `name` 是 Skill 的唯一标识符，也是对应斜杠命令名。命名只能包含小写字母、数字和连字符，且不能和内置斜杠命令冲突。
- F3: `description` 是一句话功能说明，必须出现在 `/help` 和命令补全可见信息里，也会作为自然语言自动匹配时的可见说明。
- F4: `mode` 支持 `inline` 和 `fork`，默认 `inline`。`inline` 模式共享当前对话历史，执行结果留在主历史里。
- F5: `fork` 模式以前台阻塞方式运行独立对话，主界面显示运行状态；完成后只向主对话回流一条简短总结，完整 fork 过程不进入主历史。
- F6: `context` 只在 `fork` 模式下生效，支持 `full`、`recent`、`none`。`full` 表示带入完整对话摘要，默认值为 `full`；`recent` 表示带入最近 5 条消息；`none` 表示不带入主对话上下文。
- F7: `model` 可选。指定时，Skill 调用期间临时使用该模型；调用结束后恢复当前默认模型。不指定时使用当前默认模型。
- F8: `tools` 可选。未设置或为空时不收窄工具；设置后只在本次 Skill 调用期间暴露白名单里的工具。加载 Skill 的系统级工具不受白名单限制。
- F9: 如果 `tools` 里引用不存在的工具，该 Skill 视为无效并跳过注册，同时报告警告；其他 Skill 和 LunaCode 正常运行。
- F10: Skill 按三层来源加载：项目级 `<项目>/.lunacode/skills/` 优先于用户级 `~/.lunacode/skills/`，用户级优先于编译进程序的内置 Skill。同名 Skill 只由最高优先级的有效版本生效。
- F11: 解析失败的单个 Skill 不阻断整体加载。首次解析失败且没有缓存版本时跳过该 Skill，不注册命令；已有缓存版本时执行回退到上一次成功解析的版本并给出警告。
- F12: 每次执行 Skill 时重新解析对应文件或目录入口，确保用户修改 Skill 后下次调用立即生效，无需重启 LunaCode。
- F13: 单文件 Skill 由一个 Markdown 文件表示，frontmatter 后的正文就是发给模型的 SOP prompt body。
- F14: 目录型 Skill 由一个目录表示，入口文件固定为 `SKILL.md`。目录中的示例、模板、脚本和参考文档属于附属资源，不会在加载 Skill 时全部进入上下文；Skill 正文需要说明这些资源的用途和读取时机，Agent 通过已有工具按需访问。
- F15: 本章不把目录型 Skill 的附属脚本自动注册为新工具；脚本只作为附属资源存在，需要时由 Agent 使用已有工具读取或执行。
- F16: 调用 Skill 时，将 prompt body 中所有 `$ARGUMENTS` 占位符替换为用户在斜杠命令后传入的原始参数；没有参数时替换为空字符串。
- F17: 显式斜杠调用总是执行对应 Skill。`/name 参数` 使用参数替换后执行；`/name` 无参数时也执行，只是 `$ARGUMENTS` 为空。
- F18: 普通自然语言对话也支持 Skill 发现。模型上下文中只注入可用 Skill 的 `name` 和 `description`，Agent 需要完整 SOP 时通过系统级加载工具按需加载。
- F19: 完整 SOP 只对本次 Agent 调用生效；调用完成后不作为持久激活状态继续钉在后续轮次的环境上下文里。
- F20: 自动注册 Skill 为斜杠命令时，需要参与 `/help`、命令解析和补全；如果 Skill 名称和内置斜杠命令冲突，该 Skill 无效并给出警告。
- F21: 内置样板 Skill 在本章提供 `commit` 和 `test`。现有硬编码 `/review` 命令保持不变，本章不提供内置 `review` Skill。

## 非功能需求

- N1: 单个坏 Skill 不能导致 LunaCode 整体不可用；错误应尽量局部化，并能被用户观察到。
- N2: 两阶段加载必须节省上下文：未加载 Skill 时只暴露名称和说明，目录型 Skill 的附属资源只在真正需要时读取。
- N3: Skill 调用产生的工具白名单、模型覆盖和完整 SOP 上下文必须是本次调用级别的临时状态，不能污染后续普通对话。
- N4: Skill 解析和热更新应保持交互流畅，常见数量的本地 Skill 不应让用户输入斜杠命令时明显卡顿。
- N5: Skill 文件格式应清晰、稳定，用户能通过阅读 Markdown 文件理解它会如何被调用。
- N6: fork 模式的用户可见行为必须可预测：前台阻塞、状态可见、只回流总结，不在主历史中混入完整子对话过程。

## 不做的事

- 不做 Skill 市场分发、远程安装、版本管理和依赖管理。
- 不做 Skill 自定义工具 schema 注册，也不把目录型 Skill 的脚本包装成新的模型工具。
- 不把完整 Skill SOP 长期激活到会话上下文；本章只支持本次调用级加载。
- 不实现后台并发 fork；fork 模式以前台阻塞方式执行。
- 不把现有 `/review` 硬编码命令改造成 Skill，也不提供同名内置 `review` Skill。
- 不支持复杂参数解析、模板表达式或条件渲染；`$ARGUMENTS` 只做简单字符串替换。
- 不要求 Skill 附属资源自动进入上下文，也不扫描附属资源生成摘要。

## 验收标准

- AC1: 当项目级、用户级和内置级存在同名有效 Skill 时，系统使用项目级版本；删除项目级版本后，用户级版本生效；删除用户级版本后，内置版本生效。
- AC2: 当某个 Skill frontmatter 缺少 `name` 或 `description`、`name` 格式非法、`mode` 或 `context` 取值非法时，该 Skill 被跳过并产生可观察警告，其他 Skill 仍可使用。
- AC3: 当 Skill 名称与内置斜杠命令冲突时，该 Skill 被跳过并产生可观察警告，原内置斜杠命令仍可使用。
- AC4: `/help` 和命令补全能显示有效 Skill 的命令名与描述；首次解析失败且无缓存的 Skill 不出现在 `/help` 和补全里。
- AC5: 用户输入 `/commit 重点关注安全问题` 时，实际发给模型的 Skill prompt 中所有 `$ARGUMENTS` 都被替换为 `重点关注安全问题`。
- AC6: 用户输入 `/commit` 且不带参数时，实际发给模型的 Skill prompt 中所有 `$ARGUMENTS` 都被替换为空字符串。
- AC7: 普通自然语言请求中，模型能看到可用 Skill 的名称和说明，并能通过系统级加载工具加载某个 Skill 的完整 SOP；该完整 SOP 不会持续影响下一次用户请求。
- AC8: 设置 `tools` 的 Skill 在执行期间只暴露白名单工具；调用结束后，后续普通对话恢复原本工具集合；系统级加载工具不被白名单隐藏。
- AC9: `tools` 引用不存在工具的 Skill 被视为无效，不注册为命令，不影响其他 Skill 和普通工具使用。
- AC10: 设置 `model` 的 Skill 调用期间使用指定模型；调用结束后，状态和后续请求恢复当前默认模型。
- AC11: `inline` Skill 执行时共享主对话历史，模型回复正常进入主历史。
- AC12: `fork` Skill 执行时主界面进入运行状态并阻塞新输入；完成后主历史只出现一条包含 Skill 名称、用户请求、关键结论、产物路径或后续建议的简短总结。
- AC13: `context: recent` 的 fork 调用只带入最近 5 条主对话消息；`context: none` 不带入主对话消息；未设置 `context` 时按 `full` 处理。
- AC14: 修改某个有效 Skill 文件后，下次执行该 Skill 使用新 prompt，无需重启 LunaCode。
- AC15: 已有缓存版本的 Skill 在热更新解析失败时，下一次执行回退到上一次成功解析版本并给出警告。
- AC16: 目录型 Skill 执行时只加载 `SKILL.md` 的 SOP；附属示例、模板、脚本和参考文档不会自动进入上下文，只有 Agent 按 SOP 指引使用已有工具读取或执行时才消耗 token。
- AC17: 内置样板中 `/commit` 和 `/test` 可作为 Skill 调用；现有 `/review` 仍按原内置斜杠命令工作。
- AC18: 市场分发、版本管理、自定义工具注册、后台 fork 和复杂模板表达式均不会出现在本章的用户可见能力中。
